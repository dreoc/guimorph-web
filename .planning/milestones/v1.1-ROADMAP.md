# Milestone v1.1: UI Modernization

**Status:** ✅ SHIPPED 2026-06-29  
**Phases:** 10–15  
**Total Plans:** 18  
**Requirements:** 18/18 validated (+ 3 constraints)

## Overview

In-place Tk/ttk UI modernization: themed shell, direct-manipulation controls, in-GUI feedback, guided workflow, keyboard shortcuts/undo, and Curve tab rehabilitation — without changing the C/OpenGL renderer or breaking `.dgt` round-trip (CON-01/02/03).

## Milestone Summary

**Key Decisions:**
- Modernize UI in place on Tk/ttk (reject renderer swap again)
- Workflow streamlining allowed where `.dgt` round-trip preserved
- Debounced 150ms `<Configure>` for responsive GL viewport
- Single-level undo stack cleared on nav/load
- Curve tab: inline spinboxes + batch `.redrawAllCurves` Compute path

**Issues Resolved:**
- Modal pop-ups for count entry and delete confirmation
- Blocking `tkmessageBox` flow-control gates (12 modals → inline status)
- Confusing tab-gating without explanation
- Double-click placement model invisible to users
- Console-only feedback (no status bar)
- Developer-only File menu items exposed to end users
- Curve tab half-dead (commented-out controls)
- Fixed non-resizable layout; mixed tk/ttk widgets

**Issues Deferred:**
- GPA plot blank after Compute (999.1)
- openDgt shows specimen 2 first (999.2)
- Orphaned modal count functions in dead code (Phase 11 grep gate — not user-facing)
- Per-specimen curve bind on specimen switch

**Technical Debt Accepted:**
- Some GPA spinboxes remain `tk2spinbox` (no clean ttk wrapper)
- Color swatch labels remain classic `tklabel` for background coloring
- `dgtcurvestuff` legacy `draw.curves()` path parallel to `.redrawAllCurves`
