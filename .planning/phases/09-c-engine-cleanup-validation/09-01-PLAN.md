<objective>
Satisfy CENG-03: replace the three numbered global pointer families
(`GBL_PTR_MODEL_1..5`, `GBL_PTR_CONTEXT_1..5`, `GBL_PTR_CURVE_1..6`) with fixed-capacity
static arrays whose capacities are documented as named `#define`s in the owning header
`tcl_state.h`. Also document the existing magic-number capacities on the already-array
globals (`GBL_LANDMARK_SET`, `GBL_CURVE_SET`). This is a behavior-preserving structural
rename + array-ification only (D-02) — the live engine path already uses `models[]` /
`context[]` directly, so runtime impact is near-zero.

The FIRST executable task backs up the deployed DLL (D-15) so the whole cleanup is
rollback-able before any source changes.

Purpose: Remove the numbered-global shorthand and give every fixed-capacity global a
documented limit in its header — the literal CENG-03 deliverable — without touching the
frozen `def_ZARF_9.h` type layout or the DLL interface.
Output: Updated `tcl_state.h`/`tcl_state.c`/`tcl_dispatch.c`, a fresh MSVC DLL deployed,
and the `.pre-phase9.bak` rollback artifact.
</objective>
