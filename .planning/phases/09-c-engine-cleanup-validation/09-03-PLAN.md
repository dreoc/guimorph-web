<objective>
Second half of CENG-04: remove the remaining debug cruft from the state, statistics, and
dispatch modules, then finish the macro/doc cleanup. Specifically: delete the dead
debug-dump functions in `tcl_state.c` (30 bare `printf`) and their `tcl_state.h` prototypes;
remove the 9 statistics-trace `printf` in `StatisticsFunction_ZARF_9.c` (porting any real
diagnostics to `simpleLog`); remove the 4 trace `printf` + 3 `if(0)//made inert` blocks +
duplicate `D/D1/D2/D3` macro definitions in `tcl_dispatch.c`; prune the obsolete `MAKE_INERT`
documentation in `RunTime_Defines_ZARF_9.h`; and finally delete the `D/D1/D2/D3` macro
DEFINITIONS in `def_ZARF_9.h` (safe — they are debug macros, NOT the frozen `dot_t`/enum
type layout). This plan runs after 09-02 so that ALL `D`-macro call sites (the `ogl_*` ones)
are already gone before the definitions are removed — otherwise the build breaks.

Purpose: Leave `tcl_log.c`/`simpleLog` as the single surviving diagnostic channel (D-08) and
finish CENG-04 with zero retained debug-flag facility, without touching the protected type
layout, the 8 Tcl command registrations, or `STAND_ALONE_TOOL` blocks (out of scope).
Output: Cleaned state/statistics/dispatch sources + pruned headers, fresh MSVC DLL deployed.
</objective>
