<objective>
Implement the unified `marker.c` module: a single `marker_*` core operating on a
passed-in `marker_set_t*`, two static instances `g_landmarks` / `g_anchors`, and every
existing `dot_*` / `anchor_*` / `get_*` / `*Get*` / `*Set*` name reimplemented as a
one-line wrapper forwarding the correct set (D-01/D-02/D-06). Reconcile `def_ZARF_9.h`,
swap the build (`CMakeLists.txt`: add `marker.c`, drop `dot_ZARF_9.c`), delete
`dot_ZARF_9.c` (D-07/D-09), and prove it compiles + links + exports `Tkogl2_Init` (MSVC).

The anchor asymmetry bugs (D-03) are fixed **by construction** because each set routes
through its own selection state; landmark behavior is preserved bit-for-bit (D-04); only
the in-loop `IS_IN_RANGE` select logging is trimmed (D-05).

Purpose: Collapse the duplicated dot/anchor code paths into shared marker code (CENG-02)
with zero caller changes and the lowest possible regression risk.
Output: `marker.c`, finalized `marker.h`, reconciled `def_ZARF_9.h`, edited
`CMakeLists.txt`, deleted `dot_ZARF_9.c`, a green MSVC build.

Note on coupling: `marker.c` (which defines `dot_add`, `anchor_add`, …) and
`dot_ZARF_9.c` (which also defines them) CANNOT both be in the build — that is a
duplicate-symbol link error. The CMake swap, the file deletion, and the build therefore
happen together in Task 3 of this plan, not split across plans.
