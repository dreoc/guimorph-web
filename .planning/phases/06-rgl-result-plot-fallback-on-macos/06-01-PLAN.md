<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| in-session shape data → local filesystem | `.rgl_show()` serializes the user's own GPA-result coordinates to a temp HTML widget in `tempdir()`. |
| local HTML file → default browser | `browseURL()` hands the widget path to the OS default browser (WebGL render). |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-01 | Tampering | temp widget file in shared `tempdir()` | low | mitigate | Use `tempfile(pattern = "guimorph-rgl-", fileext = ".html")` (random name) — no fixed-path clobber/symlink target. |
| T-06-02 | Elevation of Privilege / Tampering | opening the widget in a browser | low | mitigate | Use `utils::browseURL()` (no shell interpolation); never `system("open ...")`. |
| T-06-03 | Information Disclosure | widget HTML assets | low | mitigate | `saveWidget(selfcontained = FALSE)` writes only local `_files/` assets; rgl's WebGL output makes no remote CDN calls. |
| T-06-SC | Tampering | package installs (npm/pip/cargo/CRAN) | low | accept | No new package installs — `rgl`/`htmlwidgets` already installed and vetted (RESEARCH §Package Legitimacy Audit); DESCRIPTION change is a manifest edit only, so no supply-chain fetch occurs. |
</threat_model>
