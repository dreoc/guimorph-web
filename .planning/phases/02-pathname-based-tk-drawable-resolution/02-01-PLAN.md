<objective>
RND-02: make the Tk drawable reachable by frame pathname (not `winfo id`) so the
platform-native drawable is obtained through Tk's own API on every platform. This
removes the fragile 32-bit `int`->pointer cast (Phase 1 01-05 left it deliberately
in place for replacement here) that truncated the 64-bit macOS `Drawable`/`NSView`
pointer and was blamed for blank viewports on Win64. The macOS accessor is staged
behind a platform guard so Phase 4's NSGL backend has the NSWindow it needs; no GL
or macOS build work is done here.
</objective>

<tasks>
1. R bridge: in `rtkogl.R` `set("window","id", frame)`, pass the widget pathname
   (`arg1$ID`, string-safe) to `tcl("setWindow","id", framePath, -1,-2,-3)`.
2. Core resolution: in `tcl_window.c` `setWindow` "id" branch, read the pathname,
   `Tk_NameToWindow(interp, path, Tk_MainWindow(interp))`, `Tk_MakeWindowExist`,
   then `Tk_WindowId` -> per-platform accessor producing the seam `void*`.
3. Stubs: `#include <tk.h>` and `Tk_InitStubs` in `tcl_init.c`; add `USE_TK_STUBS`
   and MSVC `tkstub86.lib` in CMake; expose the Tk `X11/` shim include path.
</tasks>
