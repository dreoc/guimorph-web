<objective>
BLD-01 / BLD-02 / BLD-04: stand up the macOS toolchain and loading path so the
Objective-C Cocoa backend can compile, link, and load, without disturbing the
verified Windows build. This is build scaffolding, not rendering: the NSGL
backend is a compiling stub and Phase 4 is "first light". Three moving parts —
a tri-platform CMake that emits a Mach-O tkogl2.dylib against the frameworks, an
extension-aware .onLoad that stops loudly instead of degrading silently, and the
removal of GLUT from the draw path so macOS needs no GLUT.
</objective>

<tasks>
1. Drop GLUT from the draw path (BLD-04): in `ogl_model_ZARF_9.c` replace both
   `glutSolidSphere` with `gluSphere` on a `GLUquadric` (reuse the leaked `qobj`
   in `ogl_drawDot` and free it; one reused quadric for the downsample loop,
   freed after); guard the two `glutBitmapCharacter` label calls + a local
   `<GL/glut.h>` include to `#if defined(_WIN32)`. Remove vestigial
   `glutInitDisplayMode` from `ogl_ZARF9.c`. Platform-guard `windows.h` and the
   GL headers in `def_ZARF_9.h` and remove `<GL/glut.h>` from it.
2. macOS NSGL stub (BLD-01): add `gfx_backend_nsgl.m` (`#if defined(__APPLE__)`)
   implementing the 5 seam functions against Cocoa/NSOpenGL; take the NSWindow*,
   store contentView, no live context (Phase 4). Object pointers held as void*
   (+`__bridge`) so it builds under MRC or ARC.
3. Tri-platform CMake (BLD-01): WIN32 (WGL src + vendored headers + system libs,
   MSVC/MinGW split preserved) / APPLE (enable OBJC, add the .m, mac Tcl/Tk
   include + stub libs via `TKOGL2_MACOS_TCLTK_PREFIX`, frameworks, `.dylib`
   suffix, `MAC_OSX_TK`) / else FATAL_ERROR. Per-platform SUFFIX.
4. Generalized load (BLD-02): rewrite `.onLoad` to compute the extension from
   `[info sharedlibextension]`, search ext x arch candidates, and `stop()` with
   an accurate message on not-found and on tcl-load failure.
</tasks>
